Pilotage IA

Configuration de l'assistant IA.

L'architecture active utilise un seul appel IA : le serveur prépare les candidats locaux, l'IA propose une recommandation structurée, puis le serveur valide les indicateurs retenus.

Principe de sécurité

Le serveur reste la source de vérité

L'IA reçoit la demande utilisateur et les candidats locaux préparés par le serveur. Les indicateurs doivent exister dans le catalogue local avant d'être proposés au builder.

Appel IA unique : produit source, pays, période, thème, titre, description et indicateurs.

Validation serveur : conserve seulement les indicateurs présents dans la base locale.

Builder Django : permet à l'utilisateur de modifier la proposition avant aperçu/export.

Règles locales : valident source, pays, thème, indicateurs et limites.
